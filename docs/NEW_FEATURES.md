# 新功能说明

## 1. 下载进度条显示

### 功能描述
在下载大文件时，会在窗口右下角显示一个进度条，提供清晰的下载进度反馈。

### 特性
- **实时进度显示**：显示下载百分比和已下载/总大小（MB）
- **自动更新**：每5%更新一次进度，避免过于频繁的UI刷新
- **下载信息**：显示正在下载的URL
- **状态提示**：下载完成后自动隐藏（3秒），下载失败显示错误信息（5秒）
- **手动关闭**：可以点击关闭按钮手动隐藏进度条

### 技术实现
- 在 `plugin-api.js` 的 `downloadFile` 方法中添加了进度跟踪
- 通过 IPC 通信将进度信息发送到渲染进程
- 支持插件通过回调函数获取下载进度

### 使用示例（插件开发）
```javascript
// 在插件中使用下载功能
const downloadPath = await this.api.downloadFile(
    'https://example.com/large-file.zip',
    'my-plugin/file.zip',
    {
        onProgress: (info) => {
            console.log(`下载进度: ${info.progress}%`);
        }
    }
);
```

## 2. 大文件日志读取功能

### 功能描述
能够高效处理大体积日志文件，支持根据时间戳定位到日志的对应行数，并支持指定读取日志文件的大小。

### 特性
- **智能检测**：打开超过200MB的文件时，自动提示使用高级读取模式
- **时间戳定位**：支持根据时间戳快速定位到日志中的特定位置
- **可配置读取大小**：默认读取100MB，可自定义1-1000MB
- **多种时间格式支持**：
  - **UE格式（推荐）**: 日志 `[2025.11.04-19.19.39:123]` → 输入 `11.04-19.19.39` 或 `11.04-19.19`
  - ISO 8601: `2024-01-15T10:30:45.123Z`
  - 标准格式: `2024-01-15 10:30:45`
  - 带毫秒: `2024-01-15 10:30:45.123`
  - 斜杠分隔: `2024/01/15 10:30:45`
  - 方括号包围: `[2024-01-15 10:30:45]`
- **二分查找算法**：使用高效的二分查找算法定位时间戳位置
- **详细反馈**：显示读取的大小、起始行号、总行数等信息

### 使用方法

#### 方式1：通过对话框打开大文件
1. 打开文件时，如果文件大于200MB，会弹出提示
2. 选择"使用高级读取模式"
3. 在对话框中输入：
   - **时间戳**（可选）：例如 `2024-01-15 10:30:45`
   - **读取大小**：默认100MB，可调整
4. 点击"读取"按钮

#### 方式2：拖放大文件
1. 将大文件拖放到编辑器区域
2. 系统自动检测文件大小
3. 如果超过200MB，会提示使用高级读取模式
4. 按照对话框提示操作

### 技术实现

#### 主要功能
1. **文件大小检测**：在打开文件前检查文件大小
2. **二分查找定位**：使用二分查找算法快速定位时间戳位置
3. **分块读取**：只读取指定大小的内容，避免内存溢出
4. **行号计算**：准确计算起始行号和总行数

#### API接口
```javascript
// 在渲染进程中调用
const result = await window.electronAPI.readFileByTimestamp(
    filePath,      // 文件路径
    timestamp,     // 时间戳（可选）
    sizeMB         // 读取大小（MB）
);

// 返回结果
{
    content: "...",           // 读取的内容
    startLine: 1234,          // 起始行号
    totalLines: 5000,         // 读取的总行数
    fileSize: 524288000,      // 文件总大小（字节）
    readSize: 104857600,      // 实际读取大小（字节）
    foundTimestamp: true,     // 是否找到时间戳
    readPosition: 123456      // 读取起始位置（字节）
}
```

### 性能优化
- **内存效率**：只加载需要的部分到内存
- **快速定位**：二分查找算法，时间复杂度 O(log n)
- **分块处理**：使用8KB块进行搜索，平衡速度和精度

### 注意事项
1. 时间戳格式必须与日志文件中的格式匹配
2. 如果未找到时间戳，将从文件开头读取
3. 读取大小建议根据实际需求调整，过大可能导致内存占用高
4. 对于超大文件（>1GB），建议使用较小的读取大小（如50-100MB）

## 使用场景

### 场景1：分析特定时间段的日志
1. 打开大型日志文件
2. 输入起始时间戳，如 `11.04-10.00.00` 或 `11.04-10.00`（UE格式）
3. 设置读取大小为100MB
4. 快速定位并查看该时间段的日志

### 场景2：下载远程日志文件
1. 插件从远程服务器下载日志文件
2. 实时查看下载进度
3. 下载完成后自动打开或处理

### 场景3：处理超大日志文件
1. 对于GB级别的日志文件
2. 使用时间戳定位到关键时间点
3. 只读取必要的部分进行分析
4. 避免加载整个文件导致程序卡顿
