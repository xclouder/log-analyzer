# UE (Unreal Engine) 时间戳格式说明

## 日志格式

UE日志文件中的时间戳格式如下：

```
[2025.11.04-19.19.39:123]LogTemp: This is a log message
```

**格式说明：**
- `[YYYY.MM.DD-HH.mm.ss:mmm]`
- `YYYY` - 年份（4位）
- `MM` - 月份（2位）
- `DD` - 日期（2位）
- `HH` - 小时（2位，24小时制）
- `mm` - 分钟（2位）
- `ss` - 秒（2位）
- `mmm` - 毫秒（3位，可选）

## 用户输入格式

在LogAnalyzer中读取大文件时，你可以使用以下简化格式输入时间戳：

### 格式1：完整格式（推荐）
```
11.04-19.19.39
```
- 格式：`MM.DD-HH.mm.ss`
- 说明：月.日-时.分.秒
- 年份会自动使用当前年份

### 格式2：省略秒（简化）
```
11.04-19.19
```
- 格式：`MM.DD-HH.mm`
- 说明：月.日-时.分
- 秒会自动设置为00

## 时间戳提取规则

LogAnalyzer会自动从日志行中提取时间戳：

```javascript
// 匹配模式
[2025.11.04-19.19.39:123]  // 完整格式
[2025.11.04-19.19.39]      // 无毫秒
```

**注意：**
- 时间戳必须在方括号 `[]` 中
- 使用点号 `.` 分隔日期部分
- 使用短横线 `-` 分隔日期和时间
- 使用点号 `.` 分隔时间部分
- 毫秒部分用冒号 `:` 分隔（可选）

## 常见问题

### Q1: 为什么不需要输入年份？
**A:** 系统会自动使用当前年份。如果你需要查找去年的日志，可以使用完整格式：`2024.11.04-19.19.39`

### Q2: 可以只输入日期吗？
**A:** 不可以。必须包含时间部分。最少格式是 `MM.DD-HH.mm`

### Q3: 秒可以省略吗？
**A:** 可以。如果省略秒，系统会默认使用00秒。

### Q4: 毫秒需要输入吗？
**A:** 不需要。毫秒部分会被忽略，系统会定位到最接近的秒。

### Q5: 如果输入的时间戳不存在怎么办？
**A:** 系统会定位到最接近的时间戳位置。如果完全找不到，会从文件开头读取。

## 技术细节

### 时间戳解析流程

1. **用户输入** → `11.04-19.19`
2. **规范化** → `2025.11.04-19.19.00`（添加当前年份和默认秒）
3. **转换为Date对象** → `new Date(2025, 10, 4, 19, 19, 0)`
4. **二分查找** → 在文件中定位该时间戳
5. **返回位置** → 从该位置开始读取

### 正则表达式

```javascript
// 提取日志中的时间戳
/\[(\d{4}\.\d{2}\.\d{2}-\d{2}\.\d{2}\.\d{2}(?::\d{3})?)\]/

// 解析用户输入
/^(\d{2})\.(\d{2})-(\d{2})\.(\d{2})(?:\.(\d{2}))?$/
```

## 最佳实践

1. **使用简化格式**：如果不需要精确到秒，使用 `MM.DD-HH.mm` 格式
2. **记录关键时间点**：在分析日志时，记录关键事件的时间戳
3. **分段读取**：对于超大文件，使用时间戳分段读取
4. **结合过滤器**：读取后使用过滤器快速定位问题

```javascript
    analyzeLog(result.content);
}
```
